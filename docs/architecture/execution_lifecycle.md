# The Execution Lifecycle: Components & Artifacts

The Developer OS relies on a strict separation of concerns. To achieve this, the system uses a coordinated ecosystem of specific scripts, roles, and Markdown artifacts throughout a single Manager -> Doer -> QA cycle. 

Here is the complete inventory of all components involved in the lifecycle, ordered by the flow of execution:

## 1. The Interactive Layer (The Meeting Room)
This is where intent is gathered from the human.
*   **`projects/GEMINI.md`**: The global persona prompt for the Manager. Instructs it to act as a Tech Lead, interrogate the user, and use the dispatch tool once the "Certainty Threshold" is reached.
*   **`<target_project>/GEMINI.md`**: The "Project Brain". Contains project-specific invariants and QA rituals that the Manager references during the meeting.

## 2. The Dispatch Layer (The Desk Work)
This layer translates human intent into machine instructions without human intervention.
*   **`tools/manager_tools.py` (`implement-feature` command)**: The bridge script. It takes the conversation summary from the Interactive Manager and orchestrates the creation of contracts and the start of the execution loop.
*   **`templates/roles/manager_headless_prompt.md`**: The system prompt for the transient "Clerk" agent spawned by `manager_tools.py` to write the contracts.
*   **`gemini_cli_headless.py`**: The low-level Python wrapper used to execute the Node.js Gemini CLI in a programmatic, headless state.

## 3. The Contract Artifacts (The Flight Recorders)
These are the compiled instructions for the worker agents. They are written to the workspace and committed to the dedicated Git feature branch (e.g., `gemini-run-123`) to track the exact specifications that started the run.
*   **`IRQ.md` (Implementation Request)**: The "Space-Grade Engineering Spec" telling the Doer exactly what to build, what constraints to follow, and what is out of scope.
*   **`QAR.md` (QA Request)**: The specific validation criteria telling the QA agent exactly what risk areas to test.

## 4. The Execution Layer (The Factory Floor)
This is the isolated state machine where the actual work happens.
*   **`implementation_run.py`**: The core Python orchestrator. It manages the Amnesia Engine, tracks costs, evaluates artifacts, and creates **Time Travel Checkpoints** (Git commits) after every agent action.
*   **`run_config.json`**: Generated by the orchestrator in the Central Registry. Stores the immutable configuration (models, max iterations, amnesia settings) for the specific run.
*   **`run_state.json`**: The live telemetry file in the Central Registry. Tracks current iteration, accumulated USD cost, API errors, and the terminal status.
*   **`templates/roles/doer_prompt.md`**: The blind builder's persona. Instructs it to read the IRQ and write code.
*   **`templates/roles/qa_prompt.md`**: The strict enforcer's persona. Instructs it to read the QAR and project rituals to audit the Doer's work.

## 5. The Execution Artifacts (The Receipts)
These files prove that work was completed and verified. They are continuously overwritten in the workspace and committed to Git by the orchestrator, creating a perfect step-by-step history of the execution.
*   **`IRP.md` (Implementation Report)**: Generated by the Doer. A structured receipt detailing what was changed, deviations, and known edge cases.
*   **`QRP.md` (QA Report)**: Generated by the QA. Contains the execution checklist of rituals, the accumulated history of the run (Trajectory), and the deterministic YAML frontmatter (`outcome: final` or `outcome: to correct`) that the orchestrator parses.

## 6. The Enforcement Tooling
These tools ensure the agents play by the rules.
*   **`tools/validate_artifact.py`**: A validation utility called by the orchestrator to ensure that the generated `QRP.md` strictly adheres to the required data types and enums.
*   **`schemas/qrp_schema.json` & `schemas/irp_schema.json`**: The strict JSON schemas defining the allowed YAML frontmatter for the execution reports. If an agent violates these schemas, the orchestrator triggers a Reprimand Loop.
